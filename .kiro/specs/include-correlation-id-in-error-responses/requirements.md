# Requirements Document

## Introduction

This feature ensures every error response body includes the request/correlation ID that is already generated by the `requestId` middleware. This enables users to provide correlation IDs when reporting errors to support teams, allowing operators to quickly locate relevant server logs for debugging.

## Glossary

- **Error_Handler**: The Express middleware responsible for catching errors and formatting error responses (`src/middleware/errorHandler.js`)
- **Request_ID_Middleware**: The middleware that generates and attaches a unique identifier to each request (`src/middleware/requestId.js`)
- **Correlation_ID**: A unique identifier assigned to each HTTP request for tracing and correlation purposes
- **Error_Response**: The JSON response body returned when an error occurs
- **Response_Header**: HTTP headers included in the response message

## Requirements

### Requirement 1: Include Correlation ID in Error Response Body

**User Story:** As a user encountering an error, I want the correlation ID included in the error response body, so that I can quote it when reporting the issue to support.

#### Acceptance Criteria

1. WHEN any error occurs and an error response is generated, THE Error_Handler SHALL include the correlation ID in the response body
2. THE Error_Handler SHALL use a consistent field name for the correlation ID across all error response types
3. THE Error_Handler SHALL include the correlation ID for AppError instances
4. THE Error_Handler SHALL include the correlation ID for ValidationError instances
5. THE Error_Handler SHALL include the correlation ID for generic unexpected errors
6. THE Error_Handler SHALL include the correlation ID for 404 Not Found responses

### Requirement 2: Include Correlation ID in Response Headers

**User Story:** As a developer integrating with the API, I want the correlation ID available as a response header, so that I can access it programmatically without parsing the response body.

#### Acceptance Criteria

1. WHEN an error response is generated, THE Error_Handler SHALL include the correlation ID as an HTTP response header
2. THE Error_Handler SHALL use the header name `X-Request-ID` for consistency with the request header
3. IF the correlation context includes additional trace IDs, THE Error_Handler SHALL preserve existing correlation headers

### Requirement 3: Match Server Log Correlation IDs

**User Story:** As a support operator, I want the correlation ID in error responses to match the ID in server logs, so that I can grep logs using the ID provided by users.

#### Acceptance Criteria

1. WHEN the Error_Handler logs an error, THE system SHALL use the same correlation ID that appears in the error response
2. THE Request_ID_Middleware SHALL set the correlation ID on the request object before any error handling occurs
3. THE Error_Handler SHALL retrieve the correlation ID from the request object (`req.id`)
4. FOR ALL error logs, THE system SHALL include the correlation ID in the log metadata

### Requirement 4: Standardize Field Naming

**User Story:** As a client developer, I want a consistent field name for the correlation ID, so that I can reliably extract it from any error response.

#### Acceptance Criteria

1. THE Error_Handler SHALL use the field name `requestId` in all error response bodies
2. WHEN the error response format includes an `error` object, THE correlation ID SHALL appear as `error.requestId`
3. THE field naming SHALL remain consistent regardless of error type or status code

### Requirement 5: Verify Correlation ID Propagation

**User Story:** As a developer, I want automated tests that verify correlation ID propagation, so that I can ensure this feature works correctly across all error types.

#### Acceptance Criteria

1. WHEN tests generate error conditions, THE test suite SHALL assert that the response body contains the correlation ID
2. WHEN tests generate error conditions, THE test suite SHALL assert that the response headers contain the correlation ID
3. THE test suite SHALL verify that the correlation ID in the response matches the correlation ID in server logs
4. THE test suite SHALL cover AppError instances, ValidationErrors, generic errors, and 404 responses
