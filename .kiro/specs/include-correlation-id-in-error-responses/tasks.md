# Implementation Plan: Include Correlation ID in Error Responses

## Overview

This implementation ensures correlation IDs (already generated by requestId middleware) are consistently included in error response bodies and headers, and verified through comprehensive testing. The primary work is adding tests to validate existing behavior and adding the fast-check library for property-based testing.

## Tasks

- [ ] 1. Verify current errorHandler implementation
  - Review all error response paths in `src/middleware/errorHandler.js`
  - Confirm `requestId: req.id` is present in all error types (AppError, ValidationError, generic, 404)
  - Confirm `X-Request-ID` header is set by requestId middleware and preserved
  - Document any gaps in current implementation
  - _Requirements: 1.1, 1.2, 2.1, 2.2, 4.1, 4.2_

- [ ] 2. Add property-based testing library
  - Install fast-check as a dev dependency: `npm install --save-dev fast-check`
  - Verify installation and basic setup

- [ ] 3. Create comprehensive error response tests
  - [ ] 3.1 Create test file `tests/middleware/errorHandler-correlation-id.test.js`

  - [ ]* 3.2 Write unit tests for specific error types

    - Test ValidationError includes requestId in body and X-Request-ID header

    - Test 404 Not Found includes requestId in body and X-Request-ID header
    - _Requirements: 1.3, 1.4, 1.5, 1.6, 2.1_

## Notes

- Tasks marked with `*` are optional and can be skipped for faster MVP
- The requestId middleware already generates and sets correlation IDs, so implementation work is minimal
- Primary focus is comprehensive testing to verify existing behavior and prevent regressions
- Property tests use fast-check to validate universal correctness across randomized inputs
- Each property test runs minimum 100 iterations to catch edge cases
